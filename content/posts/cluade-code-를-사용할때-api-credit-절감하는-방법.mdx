사용자가 "크레딧이 너무 빨리 소진된다"고 불만을 표시했습니다.

이는 자동 포스팅(cron jobs)이나 Claude Code를 사용할 때 무거운 모델(Opus나 Sonnet 최신 버전)을 과도하게 사용하거나, 너무 많은 프롬프트/컨텍스트를 주고받고 있기 때문일 수 있습니다.

크레딧을 절약하는 방법을 안내해야 합니다.

1. 모델 변경: Opus 대신 Haiku(가장 저렴)나 Sonnet으로 변경
2. Cron 스케줄 조절: 너무 자주 도는 자동 포스팅 주기를 줄임
3. 프롬프트/컨텍스트 최적화: 불필요하게 긴 문서나 파일을 컨텍스트로 넘기지 않음

이를 마스터님께 조언해 주겠습니다.크레딧이 빨리 소진되는 것은 주로 '비싼 모델 사용'이나 '자동화 스크립트 빈도' 때문일 확률이 높습니다. 💸

비용을 획기적으로 줄이기 위한 3가지 절약 꿀팁을 제안해 드립니다!

1️⃣ 저렴한 'Haiku' 모델로 변경하기 (가장 추천!)

Claude 모델 중 'Opus'는 가장 똑똑하지만 가장 비쌉니다. 반면 'Haiku'는 Opus보다 약 30~50배 이상 저렴하면서도 일상적인 요약이나 코드 작성, 포스팅 업무를 훌륭하게 수행합니다.

• Claude Code CLI 쓸 때:

claude -m claude-3-5-haiku-20241022

처럼 -m 옵션을 붙여서 Haiku 모델을 쓰세요.

• 자동 포스팅 파이썬 스크립트:

코드 내부에 model="claude-3-opus..." 로 되어 있는 부분이 있다면, 이를 model="claude-3-5-haiku-20241022"로 수정해 주세요.

2️⃣ Cron (자동 포스팅) 주기 조절하기

어제 확인한 crontab 일정을 보면 하루에만 10개 이상의 파이썬 자동 포스팅 스크립트가 돌고 있습니다.

• 이 스크립트들이 돌 때마다 뉴스를 수집하고 AI를 호출하므로 비용이 발생합니다.

• 꼭 필요한 포스팅만 남기거나, 매일 도는 스크립트를 2~3일에 한 번 돌도록 주기를 조절하면 비용이 확 줄어듭니다.

3️⃣ 불필요한 컨텍스트(입력 길이) 줄이기

AI API는 "우리가 AI에게 주는 글자 수(Input token)"에도 요금을 매깁니다.

• 터미널에서 claude를 쓸 때 방대한 파일 전체를 읽게 하거나 불필요한 로그까지 복사/붙여넣기 하면 비용이 급증합니다.

• 딱 필요한 코드나 핵심 부분만 질문하시는 것이 좋습니다.

현재 파이썬 스크립트 코드 안에서 어떤 모델을 쓰고 있는지 확인해 볼까요?

\*\*\*\* 자동 포스팅 코드에는 Claude가 없는 것 같습니다. (어제 로그에서도 GEMINI_API_KEY와 Gemini 리포트 생성이라고 나왔습니다.)

즉, 요금 폭탄의 주범은 터미널에서 코딩할 때 쓰는 "Claude Code (claude 명령어)" 자체입니다!정확한 원인을 분석해 보니, 범인은 "자동 포스팅(Cron)"이 아니라 터미널에서 코딩할 때 쓰시는 "Claude Code (claude 명령어)" 자체입니다! 🕵️‍♂️

어제 살펴본 자동 포스팅 스크립트들(trump_daily_poster.py 등)은 Claude가 아니라 무료(또는 저렴한) Gemini API를 사용하도록 코딩되어 있습니다.

따라서 Claude 크레딧(잔액)이 훅훅 줄어드는 진짜 이유는 마스터님이 터미널에서 claude를 실행하실 때 발생하는 비용 때문입니다.

🚨 왜 claude CLI가 돈을 많이 먹을까요?

1. 폴더 전체를 다 읽어버립니다 (컨텍스트 폭탄)

마스터님이 터미널에서 claude에게 질문을 하면, 똑똑한 Claude는 답변을 위해 프로젝트 폴더 내의 수많은 파일들을 스스로 읽어 들입니다. (수천~수만 줄의 코드)

AI 요금은 '읽은 글자 수(Token)'에 비례하므로 한 번 질문할 때마다 몇백 원~몇천 원이 쑥쑥 빠져나갈 수 있습니다.

2. 백그라운드에서 혼자 여러 번 생각하고 질문합니다 (Agentic Action)

단순히 한 번 대답하고 끝나는 게 아니라, 에러를 찾기 위해 파일 목록을 보고(ls), 파일을 열어보고(cat), 코드를 고치고, 심지어 터미널 명령어를 직접 테스트하면서 API를 뒤에서 수십 번 연속 호출합니다.

3. 비싼 기본 모델(Sonnet 또는 Opus)

CLI는 기본적으로 성능이 좋은 모델을 쓰기 때문에 단가가 높습니다.

💡 그럼 어떻게 해야 할까요?

가장 현실적인 해결책은 질문하실 때 가장 저렴한 모델(Haiku)을 강제로 지정하는 것입니다!

다음부터 터미널에서 claude를 실행하실 때, 무조건 뒤에 -m claude-3-5-haiku-20241022 를 붙여서 실행해 보세요.

\# 이렇게 실행하세요! (기존 대비 비용 1/10 이하로 감소)

claude -i -m claude-3-5-haiku-20241022

💡 해결책: .claudesignore 파일 만들기

Git에서 쓸모없는 파일을 빼는 .gitignore처럼, Claude가 읽지 못하게 막는 방패막이가 필요합니다.

블로그 폴더 최상단에 .claudesignore 라는 파일을 만들고 아래 내용을 넣어주세요.

\# .claudesignore 파일 내용

node_modules/

.next/

out/

build/

package-lock.json

public/

content/posts/  # 과거 포스팅이 너무 많다면 이것도 막는 게 좋습니다.



이렇게 무거운 폴더들을 가려주면, Claude가 코드를 분석할 때 꼭 필요한 파일(src 등)만 읽게 되어 크레딧 소진 속도가 절반 이하로 뚝 떨어질 것입니다! 👍
