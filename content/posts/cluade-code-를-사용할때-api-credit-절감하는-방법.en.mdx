This could be because heavy models (Opus or the latest Sonnet) are being used excessively during auto-posting (cron jobs) or Claude Code usage, or because too many prompts and context are being exchanged.

Here is guidance on how to save credits.

1. Change Models: Switch from Opus to Haiku (cheapest) or Sonnet
2. Adjust Cron Schedule: Reduce the frequency of auto-posting scripts that run too often
3. Optimize Prompts/Context: Avoid sending unnecessarily long documents or files as context

Here are cost-saving tips. Credits draining quickly is most likely due to using expensive models or running automation scripts too frequently.

Here are 3 tips to dramatically reduce costs!

1. Switch to the Cheaper 'Haiku' Model (Most Recommended!)

Among Claude models, 'Opus' is the smartest but most expensive. In contrast, 'Haiku' is about 30-50x cheaper than Opus while still performing excellently for everyday tasks like summarization, code writing, and posting.

- When using Claude Code CLI:

claude -m claude-3-5-haiku-20241022

Add the -m option to specify the Haiku model.

- For auto-posting Python scripts:

If the code contains model="claude-3-opus...", change it to model="claude-3-5-haiku-20241022".

2. Adjust Cron (Auto-Posting) Frequency

Looking at the crontab schedule, over 10 Python auto-posting scripts run per day.

- Each time these scripts run, they collect news and call the AI, incurring costs.
- Keep only essential posts, or change scripts that run daily to run every 2-3 days to significantly reduce costs.

3. Reduce Unnecessary Context (Input Length)

AI APIs also charge for "the number of characters we send to the AI (Input tokens)."

- If you have Claude read entire massive files or paste unnecessary logs when using the terminal, costs spike rapidly.
- Only include the necessary code or key portions in your queries.

Why Does the Claude CLI Cost So Much?

1. It reads entire folders (context bomb)

When you ask Claude a question in the terminal, Claude reads numerous files in the project folder to answer. (Thousands to tens of thousands of lines of code)

Since AI fees are proportional to the number of characters read (Tokens), each question can cost anywhere from a few cents to several dollars.

2. It autonomously thinks and queries multiple times in the background (Agentic Action)

Instead of just answering once, it checks file lists (ls), opens files (cat), fixes code, and even tests terminal commands, calling the API dozens of consecutive times behind the scenes.

3. Expensive default model (Sonnet or Opus)

The CLI uses a high-performance model by default, so the per-unit cost is high.

So What Should You Do?

The most practical solution is to force the cheapest model (Haiku) when asking questions!

From now on, when running claude in the terminal, always append -m claude-3-5-haiku-20241022.

\# Run it like this! (Cost reduced to 1/10 or less compared to before)

claude -i -m claude-3-5-haiku-20241022

This makes it much faster and your credits will drain remarkably slower.

Why Are Credits Still Draining Fast Even with Haiku?

If credits are draining even with the cheapest model, there is only one reason:

**"Claude is reading too many files every time it thinks."**

You run it in the blog project (~/xxx.xxx.com) directory, right?

If Claude is reading and analyzing massive folders that should be ignored (e.g., node_modules, .next, hundreds of accumulated post files) every time, even the cheapest model will drain credits fast because the "number of characters read" is just too high.

Solution: Create a .claudeignore File

Just like .gitignore keeps useless files out of Git, you need a shield to prevent Claude from reading them.

Create a file called .claudeignore at the root of your blog folder and add the following:

\# .claudeignore file contents

node_modules/

.next/

out/

build/

package-lock.json

public/

content/posts/  # Block this too if you have too many past posts.



By filtering out these heavy folders, Claude will only read the files it actually needs (like src), and credit consumption will drop to less than half!
